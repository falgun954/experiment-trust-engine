# Building the Power BI Report

Power BI Desktop is Windows-only and `.pbix` is a binary format, so it has
to be assembled locally rather than generated by this repo. Everything
needed is here â€” this is a ~30-45 minute assembly job, not a build-from-scratch job.

## 1. Get the data in

Run the pipeline first so the processed tables exist:

```bash
pip install -r requirements.txt
python src/generation/generate_data.py
python src/pipeline.py
python src/load_warehouse.py   # optional, only needed if you want to query via SQL
```

In Power BI Desktop: **Get Data â†’ Text/CSV**, import:
- `data/raw/experiments.csv` â†’ rename to `DimExperiment`
- `data/raw/users.csv` â†’ `DimUser`
- `data/raw/metric_definitions.csv` â†’ `DimMetricDefinition`
- `data/processed/experiment_results.csv` â†’ `FactTrustScore`
- `data/processed/daily_lift.csv` â†’ `FactDailyLift`

Also import the metric-definition-divergence numbers by running this once
and importing the CSV it prints, or query `v_metric_definition_divergence`
directly if you loaded the SQLite warehouse:

```bash
python3 -c "
import sqlite3, pandas as pd
conn = sqlite3.connect('data/warehouse.db')
pd.read_sql('SELECT * FROM v_metric_definition_divergence', conn).to_csv('data/processed/metric_divergence.csv', index=False)
"
```
Import that as `FactMetricDivergence`.

## 2. Model relationships

- `FactTrustScore[experiment_id]` â†’ `DimExperiment[experiment_id]` (many-to-one)
- `FactDailyLift[experiment_id]` â†’ `DimExperiment[experiment_id]` (many-to-one)
- `FactMetricDivergence[experiment_id]` â†’ `DimExperiment[experiment_id]` (many-to-one)
- `DimMetricDefinition[experiment_id]` â†’ `DimExperiment[experiment_id]` (many-to-one)

## 3. Add the measures

Copy every DAX measure from `dax/measures.md` into a dedicated measures
table (create a blank table named `_Measures` to keep them organized,
standard practice).

## 4. Build the four report pages

1. **Experiment Overview** â€” Trust Score as a gauge/KPI card per
   experiment (card visual + conditional formatting: red < 60, amber
   60-84, green â‰¥ 85), table of all experiments with `Verdict` and
   `Trust Reasons`.
2. **Metric Definition Comparison** â€” clustered bar chart,
   `Active Users Def V1` vs `Active Users Def V2` by experiment, sorted by
   `Definition Agreement %` ascending.
3. **Statistical Deep-Dive** â€” line chart of `FactDailyLift[lift]` over
   `day` per experiment (shows novelty decay visually), plus a table with
   `SRM Chi Square`, `SRM P Value`, `P Value`, `CI Lower`, `CI Upper`.
4. **Metric Lineage** â€” a plain table from `DimMetricDefinition` showing
   `definition_logic`, `effective_start`, `effective_end` â€” this page is
   documentation-as-a-dashboard, on purpose.
